# 클래스

## 25.1) 클래스 정의
- 클래스 이름은 파스칼 케이스를 사용하는 것이 일반적

## 25.2) 클래스 호이스팅
- 클래스 선언문으로 정의한 클래스는 let, const 키워드로 선언한 변수처럼 호이스팅이 됨

## 25.3) 메서드
- 클래스 몸체에서 정의할 수 있는 메서드 : 생성자, 프로토타입 메서드 정적 메서드

## 25.4) 포토퍼티
### 클래스 필드 정의 제안
- 클래스 필드 : 인스턴스의 프로퍼티
- 원래 클래스 몸체에는 메서드만 선언할 수 있음
- 최신 브라우저와 최신 Node.js에서는 클래스 몸체에서 클래스 필드 정의가 가능
- 아직 ECMAScript의 정식 표준 사양으로 승급되지는 않음
- this는 클래스의 constructor와 메서드 내에서만 유효하기 때문에, 클래스 몸체에 클래스 필드를 정의하는 경우에는 this를 사용하면 안됨  
- constructor와 메서드 내에서 클래스 필드를 참조하는 경우 this를 반드시 사용해야 함
- 모든 클래스 필드는 인스턴스 프로퍼티가 되기 때문에, 필드에 함수를 할당하는 경우, 이 함수는 프로토타입 메서드가 아닌 인스턴스 메서드가 됨
### private 필드 정의 제안
- 원래 자바스크립트에서는 접근 제한자를 지원하지 않아서 인스턴스 프로퍼티는 항상 public
- 최신 브라우저와 최신 Node.js 에서는 private 필드를 정의할 수 있음
### static 필드 제안
- 원래 자바스크립트에서는 static 키워드를 사용하여 정적 메서드를 정의할 수 있었지만, 정적 필드를 정의할 수는 없었음
- 최신 브라우저와 최신 Node.js static 키워드를 이용해 정적 필드를 정의할 수 있음

## 25.5) 상속에 의한 클래스 확장
### 클래스 상속과 생성자 함수 상속
### 동적 상속
- extends 키워드는 클래스뿐만 아니라 생성자 함수를 상속받아 클래스를 확장할 수 있음
- extends 키워드 앞에는 반드시 클래스가 와야 함
### super 키워드
- super 참조가 동작하기 위해서는 super를 참조하고 있는 메서드가 수퍼클래스의 프로토타입을 찾을수 있어야 함
- 메서드의 내부 슬롯 [[HomeObject]]는 자신을 바인딩하고 있는 객체를 가리킴
### 표준 빌트인 생성자 함수 확장
- String, Number, Array 같은 표준 빌트인 객체도 생성자 함수 이므로 extends 키워드를 사용하여 확장할 수 있음
